## Issue: Explore Metaflow CLI improvements
## Issue Description:
As Metaflow supports building extensions for the client, some authors might want to extend parts of the CLI that are difficult or impossible to reliably get to. Identifying such areas in the code base and proposing ways to make them extensible would prove useful.

## Goals
* Getting familiar with the Metaflow CLI internals and Metaflow extensions structure.
  
  * Identifying non-customisable sections of the CLI.
* Propose a design for improving the Metaflow CLI in any way.
  
  * Proposal should contain considerations regarding compatibility, and extension support.

## Instructions
This issue is meant to be a venue for asking further questions on the topic. Implementation details and proposals are to be kept separate.

When you have met the goals of the issue, it is time to show and tell. You can do this for example with your own fork with an **issue** or **pull request** and messaging a mentor on Slack with a link.
